## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Stored `spec_json` text → `ResourceSpec.model_validate_json` in dispatcher | Persisted JSON crosses the boundary back into Python; malformed or tampered JSON could cause Pydantic ValidationError; Dispatcher must not swallow this silently and must not allow it to bypass status writes |
| `ADAPTER_REGISTRY[spec.kind]` lookup | A spec with an unexpected kind (e.g., a future kind not yet in the registry, or a tampered kind string) must be handled deterministically — neither raising an uncaught KeyError nor silently completing |
| Adapter result → ResourceRecord write | Adapter outputs `ProvisionResult` whose `provider_metadata: dict[str, Any]` and `activity_log: list[str]` are serialized via `json.dumps`; mock adapters generate constant strings, but the future real adapters could carry arbitrary content. `json.dumps` correctly escapes any embedded characters |
