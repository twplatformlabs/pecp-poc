Wire the Walking Skeleton — the thinnest end-to-end vertical slice that proves the entire PECP stack. Plan 01 locked the contracts (models, ABC, RequestContext, test scaffolds with three SKIPPED tests). This plan stands up SQLite-backed persistence, a live FastAPI server, the GET/POST `/resources` route pair with team-scope enforcement (ARCH-01), and a Typer `pecp apply` CLI that wraps the API. The user story drives the slice: **As a team member, I want to run `pecp apply -f resource.yaml --team payments`, so that the platform persists my YAML and I can list it back without knowing which AWS account I'm in.** Provisioning is intentionally NOT in scope — adapters arrive in Phase 2. Persistence here is the minimum needed to prove the stack is real.

ARCH-01 (team scope 400) becomes live and provable.

Output: A runnable PECP control plane — `uvicorn pecp.api.main:app` boots, `pecp apply -f example.yaml --team payments` persists a record, `pecp get --team payments` (and equivalent HTTP `GET /resources?team=payments`) lists it.
