  <action>
    Create `src/pecp/integrations/github.py` with the following structure:

    **Module-level setup:**
    - Module docstring: "GitHubIntegration — provisions GitHub teams, repos, and memberships (GH-01 through GH-05)."
    - Imports: `logging`, `uuid`, `datetime` (datetime, timezone, utc), `httpx`, `select` from `sqlalchemy.future`
    - Import IntegrationBase, MemberSnapshot, ProjectSnapshot, TeamSnapshot from `pecp.integrations.base`
    - Import `IntegrationConfig` from `pecp.integrations`
    - Module-level DB import: `import pecp.persistence.database as _db` (module reference, NOT from-import AsyncSessionLocal — avoids DetachedInstanceError from import-time session binding)
    - Module-level logger: `logger = logging.getLogger(__name__)`
    - Constant: `GITHUB_API = "https://api.github.com"`

    **Helper function `_sanitize(name: str) -> str` (D-07):**
    - Return `name.lower().replace(" ", "-")`
    - Docstring: "Lowercase and replace spaces with hyphens for GitHub API consumption (D-07)."

    **Helper function `_write_team_slug(team_id: str, slug: str) -> None`:**
    - Opens a fresh AsyncSession via `_db.AsyncSessionLocal()`
    - Selects TeamRecord by team_id (import TeamRecord from pecp.persistence.models inside function body)
    - If found, sets record.github_team_slug = slug and commits
    - If not found, logs warning with team_id
    - Docstring explains this opens a new session because the request-scoped session is closed by the time this background task runs

    **Helper function `_write_project_repo(project_id: str, repo_name: str, repo_url: str) -> None`:**
    - Opens a fresh AsyncSession via `_db.AsyncSessionLocal()`
    - Creates ProjectRepoRecord (import inside function body) with id=uuid.uuid4().hex, project_id, repo_name, repo_url, created_at=datetime.now(timezone.utc)
    - Adds and commits
    - Docstring: same session isolation rationale

    **Helper function `_fetch_team_slug(team_id: str) -> str | None` (D-05, D-06):**
    - Opens a fresh AsyncSession via `_db.AsyncSessionLocal()`
    - Selects TeamRecord by team_id
    - If found, returns record.github_team_slug
    - If not found, logs warning and returns None
    - Docstring explains this re-fetches github_team_slug to avoid race conditions where on_team_create's DB writeback hasn't completed

    **Class `GitHubIntegration(IntegrationBase)` (GH-01):**

    `__init__(self, config: IntegrationConfig) -> None`:
    - Store `self._org = config.GITHUB_ORG`
    - Create `self._client = httpx.AsyncClient(...)` with:
      - `base_url=GITHUB_API`
      - Headers dict: Authorization: `f"Bearer {config.GITHUB_PAT}"`, Accept: `"application/vnd.github+json"`, X-GitHub-Api-Version: `"2022-11-28"`
    - Docstring on class: "GitHub integration that provisions teams, repos, and memberships (GH-01). Uses httpx.AsyncClient with Bearer token auth. All errors are caught inside hooks and re-raised — fire_integrations prevents propagation."

    `async def on_team_create(self, team: TeamSnapshot) -> None` (GH-02, D-01, D-07):
    - Call `_sanitize(team.name)` to produce the sanitized name
    - try/except block inside:
      - POST /orgs/{self._org}/teams with json `{"name": sanitized_name, "privacy": "closed"}`
      - If resp.status_code == 422: Log warning with team.name, return early (D-01 — no GET-fallback, no slug writeback)
      - resp.raise_for_status() for non-422 errors
      - Extract slug: str = resp.json()["slug"]
      - await _write_team_slug(team.id, slug)
    - except clause: logger.exception(...) with team.name context, then raise

    `async def on_project_create(self, project: ProjectSnapshot, team: TeamSnapshot) -> None` (GH-03, D-01, D-07):
    - Build repo_name = _sanitize(f"{team.name}-{project.name}")
    - try/except:
      - POST /orgs/{self._org}/repos with json `{"name": repo_name, "private": False, "auto_init": False}`
      - If resp.status_code == 422: Log warning with repo_name, return early (D-01 — no GET-fallback, no repo writeback)
      - resp.raise_for_status()
      - Extract repo_url: str = resp.json()["html_url"]
      - await _write_project_repo(project.id, repo_name, repo_url)
    - except: logger.exception(...) with project.name context, then raise

    `async def on_member_add(self, user: MemberSnapshot, team: TeamSnapshot) -> None` (GH-04, D-05, D-06):
    - slug = await _fetch_team_slug(team.id) — re-fetch from DB per D-05
    - If slug is None: Log warning with team.name, return silently (D-06)
    - try/except:
      - PUT /orgs/{self._org}/teams/{slug}/memberships/{user.user_id} with json `{"role": "member"}`
      - resp.raise_for_status()
    - except: logger.exception(...) with user.user_id and team.name, then raise

    `async def on_member_remove(self, user: MemberSnapshot, team: TeamSnapshot) -> None` (GH-04, D-02, D-05, D-06):
    - slug = await _fetch_team_slug(team.id) — re-fetch from DB per D-05
    - If slug is None: Log warning with team.name, return silently (D-06)
    - try/except:
      - DELETE /orgs/{self._org}/teams/{slug}/memberships/{user.user_id}
      - If resp.status_code not in (204, 404): Only call resp.raise_for_status() for unexpected statuses (D-02 — 404 on DELETE means user already not a member, treat as success)
    - except: logger.exception(...) with user.user_id and team.name, then raise

    `async def aclose(self) -> None` (D-03):
    - await self._client.aclose()
    - Docstring: "Close the httpx.AsyncClient connection pool (D-03)."

    **PAT safety note:** Never interpolate PAT value into any log message. Only variable NAMES (team.name, user.user_id, project.name, repo_name) appear in format strings for logger.exception/warning calls. The PAT is only used in the AsyncClient Authorization header.
