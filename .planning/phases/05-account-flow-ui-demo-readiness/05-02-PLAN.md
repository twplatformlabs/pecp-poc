| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-05-02-04 | Repudiation | No audit log for seed runs | accept | PoC scope; seed reruns are inherently idempotent and traceable via git history of `scripts/seed.py`. |
| T-05-02-05 | Tampering | `sys.path.insert` could shadow installed packages | mitigate | Path inserted is `<repo>/src` — only adds the in-repo `pecp` package; harmless if `pip install -e .` already exposed the same path (no-op). |
| T-05-02-SC | Tampering | npm/pip/cargo installs | n/a | This plan adds NO new dependencies. All imports (`asyncio`, `json`, `sys`, `uuid`, `datetime`, `pathlib`, `sqlalchemy`) are already in the project. No package-legitimacy checkpoint required. |
