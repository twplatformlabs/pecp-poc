Exceptions:
- Top nav height: 56px (h-14 = 3.5rem). Tailwind's standard; not a custom token.

## Interaction Contract

### Team Dropdown (Top Nav)

- Source: `GET /teams?limit=50` via TanStack Query
- Fallback: derive unique `team` values from `GET /resources` response if `/teams` endpoint is unavailable
- Selecting a team fires `refetch()` on both Inventory and Deployments queries
- Default selection: first team alphabetically on initial load
- Placeholder while loading: "Loading teams..." disabled option
- No search / type-ahead — plain Select with ≤ 10 options expected

### Refresh Button (Top Nav)

- Label: "Refresh" with a `RefreshCw` lucide icon (16px, left of label)
- Click: calls `queryClient.invalidateQueries()` for active tab's query key
- During refetch: button shows `RefreshCw` with `animate-spin` class; button is `disabled`
- No auto-polling interval. `refetchOnWindowFocus: false`. `staleTime: Infinity` (user controls freshness).
- Source: CONTEXT.md D-09

### Tabs (Inventory / Deployments)

- Implemented with shadcn `<Tabs>` component
- Default active tab: Inventory
- Tab state: local React `useState` — not URL-driven, not persisted
- Tab switch does NOT trigger a new network fetch — data already fetched for active team is reused
- Active tab indicator: 2px bottom border in `bg-primary`

### Inventory Table

- Columns (in order): Name, Kind, Status, Environment, Project
- Column widths: Name (auto, min 160px), Kind (120px), Status (100px), Environment (100px), Project (auto, min 120px)
- Row count: no pagination for PoC — render all rows (seed data ≤ 50 rows)
- Row hover: `hover:bg-muted/50`
- No row click action — read-only
- Column headers: `text-xs font-medium text-muted-foreground uppercase tracking-wide`
- Kind cell: plain text, e.g., "PECPLambda" — no truncation
- Environment cell: plain text, lowercase, e.g., "prod"
- Project cell: plain text or em-dash ("—") if no project assigned

### Deployments Tab

- Environment filter: shadcn `<Select>` above the table
- Options: All, dev, staging, prod (in this order)
- Default: All
- Filter is client-side — does not trigger a new API call. Filters the already-fetched resource list by `environment` field.
- Same table columns as Inventory tab

### Status Badge

- Inline component: `StatusBadge({ status }: { status: string })`
- Renders a `<span>` with the color classes from the Status Badge Color Palette table above
- Text: status string as-is (no capitalization transform — match CLI output exactly)
- Unknown statuses fall back to: `bg-slate-100 text-slate-600`

## States

### Loading State

- Show skeleton rows (3 rows) inside the table body while the initial fetch is in flight
- Use shadcn `<Skeleton>` with `h-4 w-full` per cell, wrapped in `<TableRow>`
- Refresh button spins (`animate-spin` on icon) during any active fetch
- Team dropdown is disabled during loading

### Empty State

- Trigger: team selected but `GET /resources` returns an empty array
- Layout: centered vertically and horizontally in the content area
- Icon: `PackageOpen` from lucide-react, 48px, `text-muted-foreground`
- Heading: "No resources found" — `text-base font-semibold text-foreground`
- Body: "Select a different team or run the seed script to populate data." — `text-sm text-muted-foreground`
- No CTA button (read-only dashboard)

### Error State

- Trigger: network failure or non-2xx response from API
- Layout: centered vertically and horizontally in the content area
- Icon: `AlertCircle` from lucide-react, 48px, `text-destructive`
- Heading: "Failed to load resources" — `text-base font-semibold text-foreground`
- Body: "Check that the PECP API is running on port 8000, then click Refresh." — `text-sm text-muted-foreground`
- Refresh button remains visible and enabled in the top nav

### No Team Selected State

- Trigger: initial load before user selects a team (if team list load itself has not completed)
- Body: "Select a team from the dropdown to view its resources." — `text-sm text-muted-foreground`, centered

## API Integration

For executor reference — what the UI calls and where:

| Query | Endpoint | TanStack Query Key |
|-------|----------|--------------------|
| Team list | `GET /teams?limit=50` | `['teams']` |
| Inventory (all resources for team) | `GET /resources?team={team}` | `['resources', team]` |
| Deployments (filtered client-side) | Same as inventory — filter by `environment` in JS | `['resources', team]` |

CORS requirement: FastAPI must have `CORSMiddleware` allowing `http://localhost:5173`. Source: CONTEXT.md D-06.

Vite proxy: configure `vite.config.ts` with `server.proxy` to route `/api` → `http://localhost:8000` so all fetch calls use relative paths (`/api/resources?team=...`). This avoids hardcoding the API URL in the React app.
