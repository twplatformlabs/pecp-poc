## Phase Goal

**As a** platform engineer using PECP, **I want to** group resources into named projects scoped to environments and query a deployment audit trail per environment via `pecp projects` and `pecp deployments`, **so that** I have compliance-grade visibility into who changed what, where, and when across my team's resources without the soft-deleted history corrupting my live views.

<objective>
Deliver the PROJECTS + DEPLOYMENTS + SOFT-DELETE vertical slice that finishes Phase 4. This plan adds two new route modules (`/projects`, `/deployments`), surgically modifies the existing `/resources` route module to soft-delete on DELETE, filter out soft-deleted rows from list/get queries, and write `DeploymentRecord` audit rows atomically on every mutation (D-10/D-11/D-12). It wires both new routers into the FastAPI app and adds the matching CLI commands (`pecp projects`, `pecp deployments`) plus the `--project` flag on `apply` (D-07) and the `--json` backfill on `pecp get` and `pecp status` (D-17).

Purpose: This plan turns every remaining Wave 0 RED test GREEN: `tests/test_api/test_projects.py` (TEAM-02), `tests/test_api/test_deployments.py` (TEAM-03 audit + env filter), `tests/test_api/test_soft_delete.py` (D-11/D-12 invisibility), and the projects/deployments/--json-backfill CLI tests in `tests/test_api/test_cli.py` (CLI-07, CLI-08, D-17). Together with Plan 02, every Phase 4 requirement ID and success criterion is satisfied.

Output: Two new route modules, surgical modifications to `resources.py` (one new helper `_maybe_get_project_id`, three new `deleted_at IS NULL` filters, three new `DeploymentRecord` writes, two new `project` assignments), two new top-level CLI commands, one new flag on `apply`, two new flags on `get`/`status`, and two `include_router` lines in `main.py`.
