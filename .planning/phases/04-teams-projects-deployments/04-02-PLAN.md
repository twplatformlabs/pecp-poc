## Phase Goal

**As a** platform engineer using PECP, **I want to** create a team via `pecp team create payments --owner alice` and immediately query its members via `pecp team payments`, **so that** I can establish multi-team ownership boundaries on the PECP control plane before grouping any resources.

<objective>
Deliver the complete TEAMS vertical slice: a single `pecp` CLI invocation creates a team via the API, the API persists `teams` + `team_members` rows, and the CLI immediately renders the full team panel (D-13). A second `pecp team <name>` invocation reads back the same data. The 409-on-duplicate behavior (D-03) prevents accidental overwrites. The `--json` flag (D-17) on both sub-commands produces stdout-clean JSON for scripting.

Purpose: This plan turns the Wave 0 RED tests in `tests/test_api/test_teams.py` GREEN, plus the team-related extensions in `tests/test_api/test_cli.py`. Together they satisfy requirements TEAM-01, CLI-05, CLI-06 and Phase 4 success criterion 1.

Output: One new route module, one app wiring line, one new CLI sub-application with two commands and shared rendering. After this plan, teams are queryable end-to-end via both the REST API and the CLI.
</objective>
