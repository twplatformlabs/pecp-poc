<objective>
Build the structural foundation of the integration hook framework (INTG-01, INTG-02, INTG-03):
the `IntegrationBase` ABC with optional async lifecycle hooks, the three snapshot dataclasses
that hooks receive, the `NoOpIntegration` reference implementation/spy, the
`INTEGRATION_REGISTRY` list, the `fire_integrations` dispatcher with per-integration error
isolation, and the `IntegrationConfig` / `load_and_register_integrations()` env-var loader
that disables (rather than crashes) the integration when `GITHUB_PAT`/`GITHUB_ORG` are unset.
Add `pydantic-settings>=2.14` and dev `pytest-httpx>=0.36` to `pyproject.toml`. Cover all
four artifacts with unit tests in `tests/test_integrations/` — no route wiring in this plan.

Purpose: Lock the contract before any GitHub code (Phase 8) or route wiring (Plan 02) is
introduced. Mirrors the existing `AdapterBase` / `ADAPTER_REGISTRY` pattern so future
integrations (Jira, Slack) plug in without re-architecting.

Output: New `src/pecp/integrations/` package with three modules, five new test files,
two new pinned dependencies in `pyproject.toml`.
</objective>
